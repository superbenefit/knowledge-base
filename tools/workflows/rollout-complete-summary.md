## Relationship Types Documented

### 1. Pattern ‚Üí Primitive (Implementation)
**Example**: cell-state.md ‚Üí nucleus.md + circle.md
- **Forward**: Pattern explains how it uses primitive
- **Backlink**: Primitive lists patterns that implement it
- **Section**: "Implementation in Patterns"

### 2. Pattern ‚Üí Pattern (Composition)
**Example**: cell.md ‚Üí cell-state.md
- **Forward**: Higher-level pattern uses component
- **Backlink**: Component shows where it's used
- **Section**: "Related Concepts"

### 3. Any ‚Üí Lexicon (Definitional)
**Example**: All content ‚Üí governance, coordination, teams, etc.
- **Forward**: First meaningful use links to definition
- **Backlink**: Lexicon shows usage examples
- **Section**: Inline + "Related Concepts"

### 4. Primitive ‚Üî Primitive (Complementary)
**Example**: nucleus ‚Üî circle
- **Relationship**: Work together in same contexts
- **Linking**: Mutual "Related Concepts" references

### 5. Library ‚Üí Internal (Connection)
**Example**: Library entries ‚Üí lexicon + patterns
- **Purpose**: Connect external resources to internal concepts
- **Section**: "Related Concepts"

## Quality Principles Applied

### ‚úÖ DO:
- Link first meaningful use of concepts
- Add "Related Concepts" sections for exploration
- Keep link density at ~1 per 2-3 sentences in prose max
- Ensure every link genuinely aids understanding
- Make relationships bidirectional
- Use consistent section naming

### ‚ùå DON'T:
- Link every instance of a term
- Over-saturate body text
- Create circular immediate references
- Link obvious terms in context
